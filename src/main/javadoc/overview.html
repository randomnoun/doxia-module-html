The doxia-module-html project is Doxia module for parsing standard HTML files.

This project exists because none of the existing Doxia modules appear to handle standard HTML files, instead requiring documentation to be written in strict XHTML, or other markup languages such as APT, FML, Markdown, and TWiki. A list of available formats is given at the Doxia Markup Languages Reference page.

This module will add support for (possibly malformed) HTML documents; meaning that HTML will be parsed in much the same way that a HTML document is parsed by a web browser; i.e. tags such as <p> <tr> and <td> will be automatically closed.

This module uses the tagsoup library in order to convert malformed HTML into XHTML before it is parsed by the standard Doxia Sink API.

In order to use this module, you will need to include it in your project's pom.xml file. See the site documentation for a description on how to do this.
